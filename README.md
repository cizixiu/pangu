## 📖 是什么

**盘古·开天术**是一个 AI Agent Skill，能够将 IMA 知识库（或任何文档集合）蒸馏为专业、可安装的 Skill 文件。

| 对比维度 | 知识库 | 蒸馏后的 Skill |
|---------|--------|---------------|
| 使用方式 | 搜索 → 读取 → 理解 | 触发关键词 → 自动执行 |
| 知识形态 | 原始文档（散装） | 结构化流程 + 精炼规则 |
| 门槛 | 需要知道搜什么、怎么问 | 说一句话就触发完整流程 |
| 复用性 | 每次都要搜+读+理解 | 安装即用，开箱即得 |

---

## ✨ 功能特性

支持 **四种蒸馏模式**：

| 模式 | 说明 | 触发示例 |
|------|------|----------|
| 🗂️ 完整蒸馏 | 蒸馏整个知识库 | 「把 @XX知识库 蒸馏成 Skill」 |
| 📁 文件夹蒸馏 | 只蒸馏指定文件夹 | 「只蒸馏『XX文件夹』里的内容」 |
| 📄 单文档蒸馏 | 只蒸馏单篇文档 | 「把『XX文档』蒸馏成 Skill」 |
| 🔁 任务蒸馏 | 把工作流变成 Skill | 「把我刚才做的调研流程变成 Skill」 |

## 💡 最佳使用场景：IMA Copilot

本技能在 **IMA Copilot** 中使用效果最佳。

**为什么？** IMA 本身就是知识库笔记软件——你在 IMA 中积累的笔记、文档、知识库，正是本技能蒸馏的原料。整个工作流无缝衔接：

```
IMA 知识库（原始积累）
    ↓ 盘古·开天术 蒸馏
可安装的 Skill（结构化能力）
    ↓ 安装到 IMA Copilot
触发关键词 → 自动执行完整工作流
```

**在 IMA 中使用本技能的完整流程：**
1. 在 IMA Copilot 中对话，说「开天术」或「蒸馏知识库」
2. 按提示提供知识库名称和需求
3. 本技能自动读取 IMA 知识库内容并蒸馏
4. 生成可安装的 Skill 文件
5. 将 Skill 安装回 IMA，即可重复使用

> 💎 **精髓**：IMA 负责「积累知识」，盘古负责「提炼能力」，形成知识增强闭环。

## 🚀 如何使用

### 安装到 WorkBuddy / OpenClaw

1. 下载本 Skill 文件夹
2. 放入 `~/.workbuddy/skills/` 目录
3. 重启会话，说「开天术」或「蒸馏知识库」即可触发

### 最快上手

```
请把我的知识库蒸馏成一个可安装的 Skill。

知识库：@合同审查知识库
这个知识库主要解决什么问题：帮助审查合同中的风险条款
希望别人安装这个 Skill 后能做什么：上传合同后自动审查风险并生成报告
```

## 🔄 蒸馏流程（7 Phase）

```
Phase 0: 入口分流        → 判断蒸馏模式
Phase 1: 需求澄清        → 确认基本信息和适用性
Phase 1.5: 创建目录      → 建立 Skill 目录结构
Phase 2: 内容读取与分析  → 读取知识库并结构化分析
Phase 2.5: 分析确认      → 检查点，用户确认分析结果
Phase 3: 框架提炼        → 提取核心框架/流程/规则
Phase 3.5: 提炼确认      → 检查点，用户确认提炼结果
Phase 4: Skill 构建       → 组装 SKILL.md
Phase 5: 质量验证        → 6项通过标准验证
Phase 6: 双Agent精炼     → 可选，结构+用户双视角评审
Phase 7: 输出与注册      → 输出文件，提供注册指引
```

## 💡 什么时候适合蒸馏？

### ✅ 适合蒸馏的知识库

- 有明确流程/步骤（如：审查流程、部署流程）
- 有规则/规范检查（如：代码规范、设计检查清单）
- 有分类决策场景（如：选型决策、风险评估）
- 有方法论/框架（如：思维框架、分析方法）
- 有固定输出格式（如：报告模板、文档格式）

### ❌ 不适合蒸馏的知识库

- 纯数据型（价格表、产品目录）→ 建议用知识库搜索
- 需要频繁更新的实时数据 → 建议用知识库+定期更新
- 纯文献存档（无操作指导）→ 建议保留为知识库
- 需要专业资质的领域 → 建议保留为参考，人工决策

## 📋 质量验证标准

生成的 Skill 必须通过以下 6 项检查：

| 检查项 | 通过标准 |
|--------|---------|
| 核心框架数量 | 3-7 个，每个有来源证据 |
| 每个框架的局限性 | 明确写出失效条件 |
| 流程步骤清晰度 | 用户能按步骤执行 |
| 边界规则完整性 | 至少 2 条具体局限 |
| 示例充分性 | 至少 2 个具体示例 |
| 文档结构完整性 | 所有必要 section 都有 |

## 🎯 设计理念

> **蒸馏的是「能做到什么」（流程/规则/框架），不是「知道什么」（信息/文档）。

- **流程 > 知识**：能执行的流程比散装知识更有价值
- **规则 > 案例**：具体规则比一堆案例更容易泛化
- **边界 > 能力**：明确边界比夸大能力更诚实
- **精炼 > 堆砌**：3 个核心框架远好于 10 个浅薄原则
